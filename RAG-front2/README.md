# RAG Frontend 2

A modern dashboard interface for the RAG (Retrieval-Augmented Generation) system. Built with Next.js 14, TypeScript, and Tailwind CSS.

## Features

- **Chat Interface**: Interactive chat with document-based AI responses
- **Document Upload**: Support for PDF, TXT, DOCX files and direct text input
- **System Statistics**: Real-time monitoring of documents, queries, and performance
- **Multiple LLM Providers**: Support for Ollama and Google Gemini
- **Source Attribution**: View and verify document sources for each response
- **Dark Mode**: Optimized for low-light environments with high contrast

## Design System

This project follows a comprehensive design system generated by the UI/UX Pro Max workflow:

- **Pattern**: Data-Dense + Drill-Down
- **Style**: Data-Dense Dashboard
- **Colors**: Dark theme with green CTA (#22C55E)
- **Typography**: Fira Code (headings) + Fira Sans (body)
- **Effects**: Smooth transitions, hover tooltips, interactive cards

## Getting Started

### Prerequisites

- Node.js 18+ and npm/yarn/pnpm
- RAG backend running on `http://localhost:8000`

### Installation

```bash
# Install dependencies
npm install

# Start development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Building for Production

```bash
# Build the application
npm run build

# Start production server
npm start
```

## API Integration

This frontend connects to the RAG backend API at `http://localhost:8000/api/v1`. Make sure the backend is running before starting the frontend.

### Available Endpoints Used

- `GET /api/v1/health` - Health check
- `POST /api/v1/query` - Query the RAG system
- `POST /api/v1/documents/upload` - Upload files
- `POST /api/v1/documents/text` - Upload text documents
- `GET /api/v1/stats` - Get system statistics
- `DELETE /api/v1/documents/clear` - Clear all documents

## Design Guidelines

### Colors
- Primary: `#0F172A` (slate-900)
- Secondary: `#1E293B` (slate-800)
- CTA: `#22C55E` (green-500)
- Background: `#020617` (slate-950)
- Text: `#F8FAFC` (slate-50)

### Typography
- Headings: Fira Code (monospace)
- Body: Fira Sans (sans-serif)

### Interactive Elements
- All clickable elements have `cursor-pointer`
- Hover states with smooth transitions (200ms)
- Glassmorphism effects with backdrop blur
- Consistent spacing and padding

## Development

### Adding New Features
1. Create new components in `components/`
2. Add API functions in `lib/api.ts`
3. Update the main page in `app/page.tsx`
4. Test with the running backend

### Styling
- Use Tailwind CSS utility classes
- Follow the design system colors and typography
- Ensure dark mode compatibility
- Maintain WCAG AA accessibility standards
